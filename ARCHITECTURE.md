## 3. Immutable Rules (Harness Constraints)

> These rules are enforced by hooks and harness-lint. Violations block execution.

### Delegation Rules
- Shinnosuke (orchestrator) NEVER implements directly — always delegates to specialists
- Stage 1 (requirements) → Misae only
- Stage 2 (planning) → Nene only
- Stage 3 (execution) → Bo, Aichan, Bunta, Masao, Kazama
- Every phase completion requires Action Kamen review (no exceptions)

### Stage-Tool Restrictions
| Stage | Allowed | Blocked |
|-------|---------|---------|
| requirements | Read, Glob, Grep, Task | Edit, Write, Bash |
| planning | Read, Glob, Grep, Write | Edit, Bash |
| execution | All tools | — |
| completion | Read, Write, Task | Bash (destructive) |

### Agent Layer Restrictions
- Execution agents (bo, kazama) cannot call Planning/Orchestration agents directly
- Utility agents (shiro, masumi, ume) cannot call any other agents
- All agents can call actionkamen (exception by design)

### Design Decision Trigger
- 2+ approaches, architecture change, pattern break, or tech stack → Midori debate required
- Debate result must be presented to user before proceeding
