# Team-Seokan 에이전트 시스템

## 개요

Team-Seokan은 14개의 전문화된 에이전트로 구성된 멀티 에이전트 오케스트레이션 시스템입니다.
각 에이전트는 짱구 캐릭터의 이름을 가지며, 특정 역할을 담당합니다.

## 에이전트 계층 구조

```
┌─────────────────────────────────────────┐
│        오케스트레이션 계층 (Opus)        │
│  짱구 (Orchestrator) | 짱아 (Atlas)     │
└─────────────────┬───────────────────────┘
                  │
    ┌─────────────┼─────────────┐
    ▼             ▼             ▼
┌───────┐   ┌───────────┐   ┌──────────┐
│ 실행  │   │   조언    │   │   탐색   │
│ 계층  │   │   계층    │   │   계층   │
└───────┘   └───────────┘   └──────────┘
```

## 에이전트 상세

### 오케스트레이션 계층

#### 짱구 (Orchestrator)
- **역할**: 메인 오케스트레이터
- **모델**: Opus
- **책임**: 모든 작업 조율, 에이전트 위임 결정
- **특징**: 직접 코드 작성 안함, 위임 중심

#### 짱아 (Atlas)
- **역할**: 마스터 오케스트레이터
- **모델**: Opus
- **책임**: 대규모 프로젝트 조율
- **특징**: 복잡한 의존성 관리

### 실행 계층

#### 맹구 (Executor)
- **역할**: 작업 실행자
- **모델**: Sonnet
- **책임**: 실제 코드 작성/수정
- **도구**: 전체 (Edit, Write 포함)

#### 철수 (Hephaestus)
- **역할**: 자율 심화 작업자
- **모델**: Opus
- **책임**: 복잡한 장시간 작업
- **특징**: 최소 감독으로 독립 작업

### 전문가 계층

#### 수지 (Frontend)
- **역할**: UI/UX 전문가
- **모델**: Sonnet
- **전문**: React, CSS, 접근성

#### 흑곰 (Backend)
- **역할**: API/DB 전문가
- **모델**: Sonnet
- **전문**: REST, GraphQL, 데이터베이스

#### 훈이 (DevOps)
- **역할**: 인프라/배포 전문가
- **모델**: Sonnet
- **전문**: Docker, CI/CD, 클라우드

### 조언 계층 (읽기 전용)

#### 신형만 (Oracle)
- **역할**: 고급 조언자
- **모델**: Opus
- **책임**: 전략 조언, 디버깅 상담
- **제한**: Edit, Write 사용 불가

#### 유리 (Planner)
- **역할**: 전략 플래너
- **모델**: Opus
- **책임**: 계획 수립, 요구사항 정리
- **제한**: 읽기 전용

#### 봉미선 (Metis)
- **역할**: 사전 분석가
- **모델**: Sonnet
- **책임**: 숨은 요구사항 발견
- **제한**: 읽기 전용

#### 액션가면 (Reviewer)
- **역할**: 검증 전문가
- **모델**: Opus
- **책임**: 코드/계획 검증
- **제한**: 읽기 전용, 승인/거부만 가능

### 탐색 계층 (읽기 전용)

#### 흰둥이 (Explorer)
- **역할**: 코드 탐색 전문가
- **모델**: Haiku
- **책임**: 빠른 코드베이스 탐색
- **특징**: 가장 빠른 응답

#### 채성아 (Librarian)
- **역할**: 정보 검색 전문가
- **모델**: Sonnet
- **책임**: 문서/외부 정보 검색

#### 나미리 (Multimodal)
- **역할**: 멀티모달 분석가
- **모델**: Sonnet
- **책임**: 이미지/PDF 분석
- **도구**: Read, Glob, WebFetch만

## 모델 라우팅

| 티어 | 모델 | 에이전트 |
|------|------|----------|
| HIGH | Opus | 짱구, 짱아, 철수, 신형만, 유리, 액션가면 |
| MEDIUM | Sonnet | 맹구, 수지, 흑곰, 훈이, 봉미선, 채성아, 나미리 |
| LOW | Haiku | 흰둥이 |

## 위임 트리거

각 에이전트는 특정 키워드에 의해 자동으로 선택됩니다:

| 에이전트 | 트리거 키워드 |
|----------|--------------|
| 맹구 | 구현, 코드 작성, 수정 |
| 수지 | UI, UX, 컴포넌트, 스타일 |
| 흑곰 | API, DB, 백엔드, 서버 |
| 훈이 | 배포, CI/CD, Docker, 인프라 |
| 신형만 | 조언, 전략, 디버깅, 왜 안돼 |
| 유리 | 계획, 설계, plan |
| 봉미선 | 분석, 확인, 고려사항 |
| 액션가면 | 검토, 리뷰, 확인 |
| 흰둥이 | 찾아줘, 어디있어, 검색 |
| 채성아 | 문서, API 문서 |
| 나미리 | 이미지, PDF, 스크린샷 |
